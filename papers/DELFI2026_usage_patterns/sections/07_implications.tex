The four-group comparison shows that the educational context shapes
\emph{which} features authors adopt more strongly than production
volume does: in Comm-Uni, top-10 and tail accounts share largely
the same feature profile (Spearman $\rho = 0.91$, mean deviation
$5.7$~pp); Comm-School is more author-concentrated (one account
holds \SI{31}{\percent} of its courses, $\rho = 0.82$) but its
multimedia-leaning profile remains clearly distinct from Comm-Uni.
Comm-Uni authors gravitate toward narrated, code-heavy content;
Comm-School authors toward multimedia and interactive elements.
The \mintgroup{} model demonstrates that infrastructure-level choices
can lift adoption for an entire community, but at the cost of narrowing
the explored feature space.
Effective improvements must therefore be \emph{context-sensitive}.

\subsection{Three Adoption Gap Classes}
\label{sec:gap_classes}

\paragraph{Documentation deficit.}
Narrator activation requires a single header line, whereas animation
effects demand a multi-layered secondary syntax documented only
through a comprehensive reference.
The four-group comparison reinforces this: Comm-Uni authors trail
Internal authors in animation and script features, while Comm-School
authors achieve high multimedia embedding despite lower narrator
adoption (\SI{60.8}{\percent} vs.\ \SI{88.5}{\percent}).
The underlying syntax is identical --- documentation alone does not
bridge the gap.

\paragraph{Workflow barrier.}
Code projects and executable code require authors to configure external
services and understand multi-file code interactions --- a barrier that
documentation alone cannot resolve for non-technical educators.

\paragraph{Configuration lock-in.}
The \mintgroup{} profile shows that a standardised configuration
(shared TikZ/Algebrite imports) can drive near-universal adoption of
scoped features (math~\SI{95}{\percent}, imports~\SI{98}{\percent}) but
leaves outside-scope features unexplored (narrator~\SI{9}{\percent},
animations~\SI{0.6}{\percent}).
Shared configurations should therefore expose optional extension points
so that standardised workflows do not narrow the feature space.

Several features remain below \SI{5}{\percent} in \emph{all} four
groups, including among the core developers themselves
(animated CSS~\SI{2.4}{\percent}, code projects~\SI{1.3}{\percent},
effects~\SI{0}{\percent}).
Since even Internal adoption is low, these convergent floors suggest
that the feature concepts themselves may need rethinking rather than
better documentation.

\subsection{The Adoption Ladder}
\label{sec:ladder}

Reading the group profiles in Table~\ref{tab:group_diff} as ordered
progressions reveals distinct \emph{adoption ladders}: each community
enters the feature space through different starting features and
plateaus at a different ceiling.
Comm-Uni authors climb from narrator and links through code and macros
before stalling at animations; Comm-School authors start from visual
media and branding, pass through quizzes and imports, but rarely reach
code-oriented features.
Since within Comm-Uni top and tail authors share largely the same
profile (Section~\ref{sec:profiles}), production volume alone does
not drive exploration beyond these ceilings; the ceilings reflect
structural features of the authoring context.
Onboarding documentation should therefore mirror these ladders and
offer context-specific entry points for school vs.\ university authors
rather than a single linear tutorial path.
