
LiaScript is a Markdown-based language for creating interactive Open Educational Resources that runs entirely client-side, enabling courses to execute in the learner’s browser without server infrastructure. While this design supports privacy and decentralisation, it also eliminates telemetry and thus obscures how pedagogical features are used in practice.

This paper addresses that gap through a corpus analysis of 3,317 validated LiaScript courses from public GitHub repositories (March 2026), authored by 307 contributors. We introduce a didactically motivated taxonomy spanning presentation, interaction, reuse, and embedding features, and apply it to the corpus. Results show a clear adoption gradient. Presentation and reuse features are widely used, whereas interaction and embedding remain selectively adopted. A comparison of core developers, a large school-focused initiative, and the broader community reveals distinct usage profiles. Notably, prolific authors do not exhibit broader feature use. We identify three adoption gap classes: documentation deficits, workflow barriers, and configuration lock-in, and derive implications for future development.
